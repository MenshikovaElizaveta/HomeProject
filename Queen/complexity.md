Для переборного решения (enumeration)

- for a in itertools.permutations(range(n)) Перебирает n! вариантов перестановок
- for i in range(n-1) Для каждого варианта перестановки пробегается по n-1 расположению ферзей
- for j in range(i + 1, n) Для каждого iтого ферзя пробегается по n-i оставшихся ферзей (в сумме = n/2)

Оценка сложности: (n!)*(n-1)*n/2


Для рекурсивного решения (recursion)

Рекурсия (f(r + 1, p)) в цикле (for c in range(n)) создает свою ветку проверок для каждого r-ферзя из столбца c и строки r. Для каждого столбца c в строке r запускается цикл проверок (if all(p[i] != c and abs(p[i] - c) != r - i for i in range(r))).  Если все проверки были бы положительны, получилось бы n**n исходов, но после проверок часть вариантов отсекается, и я не совсем понимаю как это учесть.

Попытка учесть отбрасывание по строкам:
Переменная r - номер строки, в которую подставляем ферзя. r глубина рекурсии. на каждом уровне r есть n! / (n-r)! вариантов как можно поставит r ферзей в r строк, чтобы они стояли в разных столбцах.

Оценка сложности: ∑( n! / (n-r)! ) для r от 0 до n
